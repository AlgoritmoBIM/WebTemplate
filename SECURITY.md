# Security Policy

## Reporting Security Vulnerabilities

Si descubres una vulnerabilidad de seguridad en WebTemplate, por favor reportarla responsablemente.

**No abras un issue público** si encuentras una vulnerabilidad. En su lugar:

1. Envía un email a: `security@webtemplate.dev` (o usa GitHub Security Advisory)
2. Incluye:
   - Descripción de la vulnerabilidad
   - Pasos para reproducirla
   - Posible impacto
   - Versiones afectadas

## Proceso de respuesta

1. Confirmaremos la vulnerabilidad en un plazo de 48 horas
2. Trabajaremos en un fix en una rama privada
3. Prepararemos un parche de seguridad
4. Coordinaremos el lanzamiento con los reportes

## Divulgación responsable

Agradecemos tu paciencia durante este proceso. Solicitamos que:

- No compartas la vulnerabilidad públicamente antes de que se lance el fix
- Esperes al menos 90 días antes de divulgar públicamente
- Trabajes con nosotros para resolver el problema

## Versiones soportadas

| Versión | Estado          |
|---------|-----------------|
| 1.0.x   | En soporte      |
| < 1.0   | Sin soporte     |

Recomendamos usar siempre la última versión.
